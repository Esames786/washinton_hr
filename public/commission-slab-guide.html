Commission Slab Policy — Admin Guide

HR Portal  |  hellotransport.com  |  Effective: June 2026

Overview

The commission slab system rewards agents progressively — the higher the USD profit they generate in a payroll period, the higher the percentage (or flat PKR amount) they earn. Each agent is linked to one commission setting; the setting may either be a classic single-value rule or the new slab-based rule.

Backward compatible: All existing commission settings and employee assignments are unchanged. The slab system activates only for settings created with "Slab Based" enabled.

Official Commission Slab Policy

Applies to percentage-based slab plans. Profit is the total confirmed USD profit from order_payments for the payroll period.

Calculation Examples

Example 1 — Percentage Slab

Agent earns $1,800 USD profit in June. Exchange rate = PKR 280.

Profit in USD → $1,800 → falls in Slab 4 ($1,501–$2,000) → rate = 25%

PKR Profit = 1,800 × 280 = PKR 504,000

Commission = 504,000 × 25% = PKR 126,000

Example 2 — Fixed PKR Slab

Agent earns $700 USD profit. Fixed slab plan: $501–$1,000 = PKR 6,000.

Profit in USD → $700 → falls in tier $501–$1,000 → flat amount = PKR 6,000

Commission = PKR 6,000 (no currency conversion — amount is already PKR)

Example 3 — Classic single-value (no slabs)

Old-style setting: 20% with is_slab_based=0. Agent earns $500 profit, rate = PKR 280.

PKR Profit = 500 × 280 = PKR 140,000

Commission = 140,000 × 20% = PKR 28,000

How to Create / Edit a Slab Commission Setting

1
Go to Admin → Commission Settings from the sidebar.
2
Click Add Commission Setting (or the edit pencil for an existing one).
3
Fill in Title and choose Commission Type (Percentage or Fixed).
4
Check the "Slab Based" checkbox. The single Value field hides and the Commission Slabs table appears.
5
Click + Add Row for each bracket. Fill in Profit From, Profit To (leave blank for open-ended top tier), and Value (% rate or flat PKR).
6
Set Target Type (Monthly), Status (Active), and click Save.
7
Go to Admin → Employees → Edit Employee and assign the new commission setting in the Commission Setting dropdown. Only employees with account type Commission Only or Salary + Commission will receive commission in payroll.
Editing a slab setting: When you save an update, all existing slab rows for that setting are replaced with the newly submitted rows. Always re-enter all rows when editing.

How Slab Commission Appears in Payroll

  • When payroll is generated, the system fetches all confirmed + unpaid orders from order_payments for the agent's agent_id within the payroll period.
  • Total USD profit is summed → matched against the employee's commission setting slabs.
  • The matched slab's rate (%) or flat PKR value is applied to calculate the commission amount.
  • After payroll is generated, those orders are marked is_paid = 1 to prevent double-counting in the next period.
  • The commission amount appears as an earning line "Commission from Orders" in the payslip.
  • If the agent's total profit falls outside all defined slab ranges → commission = PKR 0.

Admin Role & Permissions

Does the existing role setup cover the new features?

Yes — with one action needed for non-superadmin roles.

RoleAccessAction Needed
Superadmin (role_id = 1) Full access — all routes bypass permission check None
Other admin roles Checked against Spatie permissions table Run Sync Permissions for the role, then assign commission_settings permissions

Why no new routes were added

The slab system uses the exact same commission_settings resource routes that already existed (index, store, edit, update, destroy). Slab data (the rows) is submitted as part of the same form POST — not a separate endpoint. So no new permission entries are required.

How to sync permissions for a non-superadmin role

1
Go to Admin → Roles and click the role you want to update.
2
Click Sync Permissions (runs POST /admin/permissions/store) — this scans all current routes and adds any new ones to the permissions table.
3
Tick the commission_settings.* permissions (index, store, edit, update, destroy, assign_roles_index, assign_roles) and click Assign Permissions.
Note: If the role already had commission_settings permissions assigned before, no action is needed — the slab feature works through the same routes.
